# Export Stripping

## Type and Purpose

- **Type**: command (build-time transformation)
- **Purpose**: Remove all ES module export syntax from bundled output so GAS can parse the code

| Spec | Section |

## Business Rules

1. `export function`, `export async function`, `export const`, `export let`, `export var`, `export class`, `export abstract class` at the start of a line must have the `export` keyword removed.
2. `export { ... };` aggregation blocks (generated by Vite library mode) must be removed entirely.
3. `export default expression` must have the `export default` prefix removed.
4. Indented `export` (not at line start) must NOT be modified — these are inside string literals or other contexts.
5. Processing happens in `generateBundle` hook, after all other plugins have finished.
6. Each chunk in the bundle is processed independently.
7. Tree-shake protection markers (`/* __GAS_GLOBALS__ */ globalThis.__gas_keep__ = [...]`) are removed before export stripping runs.
8. Output is trimmed of trailing whitespace and given a single trailing newline.

## Inputs and Outputs

**Inputs**:
- `chunk.code`: string — bundled JavaScript code from Vite build

**Outputs**:
- **Success**: `chunk.code` with all export syntax and tree-shake markers removed, trailing whitespace trimmed, single trailing newline
- **Error**: N/A (regex replacements are safe; no error path)

## Error Mapping

No error conditions — transforms are pure string operations that always succeed.

## Dependencies

- `packages/gas-vite-plugin/src/transforms.ts`: provides `stripExportKeywords()` and `removeExportBlocks()`

## Endpoints / Interfaces

| Method | Path/Interface | Description |
|--------|---------------|-------------|
| `stripExportKeywords(code: string): string` | `transforms.ts` | Removes `export` keyword from inline declarations (function, async function, const, let, var, class, abstract class) |
| `removeExportBlocks(code: string): string` | `transforms.ts` | Removes `export { ... }` blocks and `export default` prefix |

## Persistence Touchpoints

Modifies `chunk.code` in-memory during `generateBundle`. The result is written to `dist/` by Vite's normal output pipeline.

## Change Impact

- Modifying regex patterns in `transforms.ts` can break all GAS output. Always run both unit and integration tests.
- Adding new export syntax support requires new regex patterns and test cases.
- The `generateBundle` hook also removes `__GAS_GLOBALS__` markers — changes to the marker format must be coordinated with the `transform` hook in `index.ts`.
- Vite version upgrades may change the format of generated `export { ... }` blocks — verify with integration tests.
